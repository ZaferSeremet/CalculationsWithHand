# ✋ Hand Calculator – Real-Time Handwritten Math Recognition

A computer vision application that lets you **draw math equations in the air with your hand** and have them solved in real-time using a custom-trained CNN model.

## How It Works

1. **Hand Tracking** – MediaPipe detects your hand and tracks finger positions in real-time.
2. **Air Drawing** – Raise your index finger to draw on a virtual canvas overlaid on the camera feed.
3. **Character Segmentation** – The drawing is converted to binary, contours are extracted, and each character is isolated using bounding boxes.
4. **CNN Recognition** – Each 28×28 segment is fed into a custom-trained CNN that classifies it as a digit (0–9) or an operator (+, −, ×, ÷).
5. **Evaluation** – The detected expression is evaluated and the result is displayed on screen.

Additionally, the **Gemini API** can be used as a fallback for complex or ambiguous expressions.

## Features

- ✍️ Draw math expressions using hand gestures (via webcam)
- 🧠 Custom CNN model trained on your own handwriting
- 🔄 **Interactive correction mode** – teach the model from its mistakes
  - **Merge** fragmented segments (e.g., `÷` detected as three parts → join into one)
  - **Split** merged characters (e.g., `97` detected as one blob → split into two)
- 🔁 **One-key retraining** – retrain the model on the fly with corrected data
- ☁️ Gemini API integration for complex expressions

## Controls

| Key | Action |
|-----|--------|
| **Draw** | Raise index finger only |
| **Pause** | Raise index + middle finger |
| `a` | Analyze with local CNN model |
| `s` | Send to Gemini API |
| `c` | Clear canvas |
| `d` | Enter correction mode |
| `r` | Retrain model with corrections |
| `q` | Quit |

### Correction Mode Keys (after pressing `d`)

| Key | Action |
|-----|--------|
| `Enter` / `Space` | Confirm prediction |
| `0`–`9` | Correct digit |
| `p` / `+` | Plus |
| `m` / `-` | Minus |
| `x` / `*` | Multiply |
| `b` / `/` | Divide |
| `j` | **Join** – merge this segment with the next |
| `k` | **Split** – cut this segment in half |
| `Esc` | Cancel correction |

## Installation

### Prerequisites
- Python 3.10+
- A webcam

### Setup

```bash
# Clone the repository
git clone https://github.com/YOUR_USERNAME/CalculationsWithHand.git
cd CalculationsWithHand

# Create a virtual environment
python -m venv venv

# Activate it
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# Install dependencies
pip install opencv-python numpy mediapipe Pillow google-genai tensorflow
```

### Configuration

1. **Get a Gemini API key** (optional, for the `s` key feature):
   - Go to [Google AI Studio](https://aistudio.google.com/apikey)
   - Generate a free API key
   - Replace `YOUR_API_KEY_HERE` in `main.py` with your key

2. **Prepare your dataset** in the following folder structure:
   ```
   dataset/
   ├── 0/        # Images of handwritten "0"
   ├── 1/        # Images of handwritten "1"
   ├── ...
   ├── 9/
   ├── divide/   # Images of "÷"
   ├── minus/    # Images of "−"
   ├── mult/     # Images of "×"
   └── plus/     # Images of "+"
   ```
   Each subfolder should contain 28×28 grayscale PNG images.

3. **Train the model**:
   ```bash
   python train_model.py
   ```

4. **Run the application**:
   ```bash
   python main.py
   ```

## Model Architecture

```
Input (28×28×1 grayscale)
    ↓
Conv2D(32 filters, 3×3) + ReLU
    ↓
MaxPooling2D(2×2)
    ↓
Conv2D(64 filters, 3×3) + ReLU
    ↓
MaxPooling2D(2×2)
    ↓
Flatten
    ↓
Dense(128) + ReLU
    ↓
Dense(14) + Softmax → [0-9, ÷, −, ×, +]
```

## Self-Learning Correction System

What makes this project unique is the **interactive feedback loop**:

1. Draw an equation → model makes predictions
2. If a character is wrong, press `d` and correct it
3. Corrected samples are saved directly to the training dataset
4. Press `r` to retrain the model with the new data
5. The model improves over time with **your specific handwriting style**

The correction mode also supports:
- **Merging** (`j`): Combine multiple segments that should be a single symbol (e.g., the dots and line of a division sign ÷)
- **Splitting** (`k`): Separate a single segment that contains multiple characters (e.g., "97" recognized as one unit)
